## Details
This command will clone the Yarn repository into a temporary folder, then build 
it. The resulting bundle will then be copied into the local project.

By default, it also updates all contrib plugins to the same commit the bundle is 
built from. This behavior can be disabled by using the `--skip-plugins` flag.

## Options
| Definition | Description |
| ---------- | ----------- |
| <h4 id="options-path%20%230" className="header-code"><code className="language-text">--path #0</code></h4> | The path where the repository should be cloned to |
| <h4 id="options-repository%20%230" className="header-code"><code className="language-text">--repository #0</code></h4> | The repository that should be cloned |
| <h4 id="options-branch%20%230" className="header-code"><code className="language-text">--branch #0</code></h4> | The branch of the repository that should be cloned |
| <h4 id="options-plugin%20%230" className="header-code"><code className="language-text">--plugin #0</code></h4> | An array of additional plugins that should be included in the bundle |
| <h4 id="options-n%2C-dry-run" className="header-code"><code className="language-text">-n,--dry-run</code></h4> | If set, the bundle will be built but not added to the project |
| <h4 id="options-no-minify" className="header-code"><code className="language-text">--no-minify</code></h4> | Build a bundle for development (debugging) - non-minified and non-mangled |
| <h4 id="options-f%2C-force" className="header-code"><code className="language-text">-f,--force</code></h4> | Always clone the repository instead of trying to fetch the latest commits |
| <h4 id="options-skip-plugins" className="header-code"><code className="language-text">--skip-plugins</code></h4> | Skip updating the contrib plugins |
